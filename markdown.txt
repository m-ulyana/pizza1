## Онлайн оплата заказа123

**Цель:** [Я как пользователь хочу оплатить свой заказ онлайн, чтобы не платить курьеру наличными](https://ulyanasm.atlassian.net/browse/PIZZA-17)

**Акторы:** пользователь, сайт, платежная система

**Основной сценарий:**  

1. Пользователь выбирает вариант оплаты “Картой онлайн” и нажимает “Оплатить”
2. Сайт перенаправляет пользователя на страницу платежной системы
3. Платежная система открывает страницу с полями для [ввода реквизитов банковской карты](https://ulyanasm.atlassian.net/browse/PIZZA-19)
4. Пользователь вводит реквизиты банковской карты и нажимает кнопку “Оплатить”
5. Платежная система отправляет пользователю по смс пароль для [подтверждения оплаты](https://ulyanasm.atlassian.net/browse/PIZZA-20) и открывает страницу для ввода этого пароля
6. Пользователь вводит пароль
7. Платежная система выводит сообщение об успешном завершении оплаты и отсылает электронный чек на электронную почту пользователя

**Исключения:**

*Неверные формат реквизитов*

4.Пользователь вводит реквизиты карты в неверном формате. Кнопка “Оплатить” не активируется

*Неверные реквизиты банковской карты*

7.Платежная система выводит сообщение об ошибке реквизитов банковской карты

*Неверный пароль-подтверждение*

7.Платежная система выводит сообщение об ошибке пароля

*Недостаточно средств для оплаты заказа*

7.Платежная система выводит сообщение о том, что на банковской карте недостаточно средств для оплаты заказа

*Не пришел пароль*

Не пришел пароль-подтверждение по смс
